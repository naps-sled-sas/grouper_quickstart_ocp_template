## Overview

This repository contains Kustomize-based OpenShift templates for deploying Grouper's quickstart mode in a containerized environment. The deployment uses a PostgreSQL database backend and demonstrates environment-specific customizations through Kustomize overlays.

## Architecture

The repository follows Kustomize's base + overlays pattern:

- **base/**: Contains the core deployment manifests
  - **database/**: PostgreSQL deployment, service, and PVC (1GB storage)
  - **grouper/**: Grouper application deployment, service, and route
  - Root `kustomization.yml` generates two secrets (postgresql credentials and grouper config)

- **overlays/**: Environment-specific customizations
  - **production/**: Patches the PostgreSQL deployment to use 2 replicas for redundancy

The Grouper application connects to PostgreSQL via the internal service name `postgresql:5432`. Both components reference credentials from the `postgresql` secret generated by Kustomize.

Grouper runs on HTTPS (port 8443) with a self-signed certificate. The route uses passthrough TLS termination, allowing the Grouper container to handle SSL directly.

## Common Commands

### Deploy to OpenShift

Create a new project and deploy base resources:
```bash
oc new-project grouper
oc apply -k base
```

Deploy with production overlay (2 PostgreSQL replicas):
```bash
oc apply -k overlays/production
```

### Access the Application

1. Get the route URL:
```bash
oc get route grouper -o jsonpath='{.spec.host}'
```

2. Retrieve the login password:
```bash
oc get secret grouper -o jsonpath='{.data.groupersystem-quickstart-pass}' | base64 -d
```

3. Access via HTTPS at: `https://<route-host>/grouper`
4. Log in with username `GrouperSystem` and the retrieved password
5. Accept the self-signed certificate warning in your browser

### Troubleshooting

View logs:
```bash
oc logs -l app=grouper
oc logs -l app=postgresql
```

## Important Notes

- The Grouper container uses the `ui` argument to run the web interface
- Hardcoded credentials are in `base/kustomization.yml` (TODO: generate random passwords)
- PostgreSQL uses RHEL9-based image: `registry.redhat.io/rhel9/postgresql-16`
- Grouper uses version 7.1.0 from Docker Hub: `docker.io/i2incommon/grouper:7.1.0`
- Grouper runs HTTPS only on port 8443 with a self-signed certificate (HTTP is disabled)
- The route uses passthrough TLS termination to allow Grouper to handle SSL
- Both PostgreSQL and Grouper containers include liveness and readiness probes for proper health monitoring
- Grouper v5+ removed Apache HTTP Server; deprecated environment variables: `GROUPER_RUN_APACHE`, `GROUPER_RUN_TOMCAT`, `GROUPER_RUN_TOMEE`
- `GROUPER_AUTO_DDL_UPTOVERSION=7.*.*` enables automatic database schema creation/updates for Grouper 7.x on first launch
- `GROUPER_UI_GROUPER_AUTH=true` enables Grouper's built-in local authentication (username/password login)
- An init container waits for PostgreSQL to be ready before starting Grouper to avoid race conditions
